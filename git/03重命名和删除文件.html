# 03，重命名和删除文件

# 1，重命名

正常情况下，对于已经添加到暂存区的文件，如果对其重命名后，

mv readme.md readme
git status
1
2

所以之后就需要，

git add readme
git rm readme.md

git status
1
2
3
4

git 提供了一个命令用于重命名，工作区和暂存区的都会被修改了，可以直接 commit 提交。

git mv readme readme.md
1

# 2，删除文件

rm 2.txt
git status
1
2

以上命令只会在工作区产生修改，暂存区中并没有更改，所以此时无法 commit 提交。还是需要将此次更改添加到暂存区后，才能提交。

而使用

git rm 2.txt
git status
1
2

工作区和暂存区的都被修改了，可以直接 commit 提交。
